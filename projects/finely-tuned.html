Finely Tuned

About

Finely Tuned is a game where you tune instruments and objects from a violin to a watch, and from a piano to a fish. Each instrument is a level and completing all the minigames in that level allows the player to move on to the next. However, to fit the game's theme, the final level of the game is designed to be impossible. This was because the game is entirely based on the pun "you can tune a piano but you cannot tuna fish" and, thus, you cannot finish the final level The Fish. Finally, after letting the player try their best to play impossible minigames, the game will conclude and explain itself

What I Did

I worked on most of the code for this game. This includes the code for the minigame system, most minigames, and the system of transitioning between levels.

Starting with the minigames, most minigames were designed to be easily changed through parameters to make it simple to make multiple levels. For instance, one game I made was a timing minigame, where the player would need to click the screen or hit the spacebar when a cursor was within a target. This game contained parameters for the cursor's size, the cursor's movement speed, and the range of the target to allow every Unity instance of the game to have a different difficulty level. The same was true for the text-based rpg minigame which used objects to contain different dialogue paths and could be swapped out for different versions of the game.

The minigame system was also designed to make it easy to add in new minigames. All minigames extended the abstract class TuningGame, which standardized opening and closing games, as well as provided a method to all minigames games to declare when they have been completed.

Finally, I also worked on the main game loop, which contained a list of all instruments and would swap instruments when all of the current one's minigames have been completed. This system, combined with the previous one, allowed the game to be created in an object-oriented manner, where every instrument could contain any number of TuningGames and would know when they have been completed. This design makes the game easy to update in the future, if I ever decide to return to the project

And, if you play the game, I hope you enjoy it!
