Abobabica: A Summer Internship I Won't Forget About

About

This is a visual novel in the style of Phoenix Wright: Ace Attorney where you play as an intern who is visiting the alien planet Abobabica. Although this is a short demo of the game, the game was meant to consist of two main elements: selecting 5 pieces of evidence while interacting with the aliens and using these pieces of evidence to answer questions. The main systems that were developed for this demo were the dialogue and evidence systems.

What I Did

I worked on all of the code for this game. The project had two notable features that I worked on: a parser and file system for determining the game's story script, as well as its use of design patterns and interfaces to make an extendable system.

First, one accomplishment from this project was the creation of a new file type used to declare what dialogue will be spoken in the game. The parser would read the file line by line and add definitions for character data and dialogue chunks. Every object from this file would contain its own unique ID that would be used to either link dialogues or declare which character was speaking in a given message. The use of IDs allowed for a dialogue system that could reuse character data and have branching dialogue paths that can loop back to previously seen dialogue chunks. The parser also ignored lines that started with “//” or were entirely blank to allow for the file to contain comments and be more readable. However, although this was a fun system to have developed, if I could restart this project today I would opt to use the json file format with a premade json parser. This is because all data stored in this file were short enough that they would have been more readable in a json file format than in a custom one.

Second, the game systems made use of design patterns and object-oriented design to be easily extendable. The class used for displaying messages used the command-design pattern to understand the effect of tags in the dialogue. For this demo, the only tag used was a “gain evidence” tag, which added evidence to the player's inventory. That being said, a dictionary was used to store the effect of all tags, making it easy to add a new one.

If you play it, I hope you enjoy the short piece of the adventure!
